# Supabase 数据库配置说明

> 影月影视 AI智能客服系统 v5.0

---

## 1. chat_records 表（已有）

存储所有访客对话记录。

| 字段名 | 类型 | 设置 |
|--------|------|------|
| id | int8 | Primary Key, Auto Increment |
| visitor_id | text | Not Null |
| role | text | Not Null ('user' 或 'ai') |
| content | text | Not Null |
| created_at | timestamptz | Default: now() |

---

## 2. human_replies 表（人工接管功能必需）

用于人工接管功能的消息传递。管理员通过此表向访客发送人工回复，系统通过此表传递接管/释放指令。

### 创建SQL

```sql
-- 创建 human_replies 表
CREATE TABLE human_replies (
  id bigint GENERATED BY DEFAULT AS IDENTITY PRIMARY KEY,
  visitor_id text NOT NULL,
  content text NOT NULL,
  is_read boolean DEFAULT false,
  created_at timestamptz DEFAULT now()
);

-- 创建索引（提高查询性能）
CREATE INDEX idx_human_replies_visitor_id ON human_replies(visitor_id);
CREATE INDEX idx_human_replies_is_read ON human_replies(is_read);
CREATE INDEX idx_human_replies_created_at ON human_replies(created_at);

-- 启用RLS（如需）
ALTER TABLE human_replies ENABLE ROW LEVEL SECURITY;

-- 允许匿名插入（客服组件需要）
CREATE POLICY "Allow anonymous insert" ON human_replies
  FOR INSERT WITH CHECK (true);

-- 允许匿名查询（客服组件轮询需要）
CREATE POLICY "Allow anonymous select" ON human_replies
  FOR SELECT USING (true);

-- 允许匿名更新（标记已读需要）
CREATE POLICY "Allow anonymous update" ON human_replies
  FOR UPDATE USING (true);
```

### 表结构

| 字段名 | 类型 | 说明 |
|--------|------|------|
| id | int8 | 主键，自增 |
| visitor_id | text | 访客ID，关联 chat_records |
| content | text | 回复内容或系统指令 |
| is_read | boolean | 是否已读取（默认false） |
| created_at | timestamptz | 创建时间 |

### 特殊 content 值（系统指令）

| content 值 | 含义 | 触发方 |
|------------|------|--------|
| `[SYSTEM:ADMIN_TAKEOVER]` | 管理员接管对话 | 后台管理员 |
| `[SYSTEM:ADMIN_RELEASE]` | 管理员释放对话 | 后台管理员 |
| `[SYSTEM:USER_REQUEST_HUMAN]` | 用户请求人工 | 访客点击按钮 |
| `[客户消息] xxx` | 人工模式下的客户消息 | 客服组件自动 |

### 工作流程

```
1. 用户点击"转人工客服"按钮
   → 插入 {visitor_id, content: '[SYSTEM:USER_REQUEST_HUMAN]', is_read: false}
   → 发送飞书通知提醒管理员

2. 管理员在后台点击"接管对话"
   → 插入 {visitor_id, content: '[SYSTEM:ADMIN_TAKEOVER]', is_read: true}
   → 访客端轮询检测到，切换为人工模式

3. 管理员发送人工回复
   → 插入 {visitor_id, content: '回复内容', is_read: false}
   → 同时插入 chat_records {visitor_id, role: 'ai', content: '[人工] 回复内容'}
   → 访客端轮询检测到，显示人工回复

4. 访客在人工模式下发送消息
   → 插入 chat_records {visitor_id, role: 'user', content: '消息'}
   → 同时插入 human_replies {visitor_id, content: '[客户消息] xxx', is_read: false}
   → 后台实时显示客户消息

5. 管理员点击"释放对话"
   → 插入 {visitor_id, content: '[SYSTEM:ADMIN_RELEASE]', is_read: true}
   → 访客端轮询检测到，恢复AI模式
```

### 客服组件轮询逻辑

```javascript
// 每3秒检查一次
setInterval(checkHumanReplies, 3000);
```

## 3. 配置步骤

1. 登录 Supabase 后台：https://supabase.com
2. 进入 SQL Editor
3. 复制上面的 SQL 语句执行
4. 确认表已创建（Table Editor 中查看）
5. 确认 RLS 权限已配置（如需要）

---

## 4. 权限说明

由于客服组件运行在外部网站（凡科）上，无法使用认证方式访问 Supabase，因此需要允许匿名访问：

- 在 Supabase Project Settings → API → Project URL 确认地址
- 在 Authentication → Policies 中确保 `human_replies` 表允许匿名 CRUD
- 或者关闭 RLS（仅测试环境，生产环境建议配置适当权限）
