### 2.2 技术选型说明

| 层级 | 技术 | 选择理由 |
|------|------|---------|
| 前端托管 | GitHub Pages | 免费、自动部署、绑定自定义域名 |
| 数据库 | Supabase | 免费额度充足、提供REST API（凡科沙盒可用）、无需后端服务器 |
| AI模型 | Moonshot AI (Kimi) | 中文理解强、价格便宜、API稳定、8k上下文足够 |
| 消息通知 | 飞书Webhook | 免费、实时推送、手机端可接收 |
| 前端框架 | 纯原生JS | 不需要框架，减少体积，直接嵌入任何网站 |
| 数据存储 | localStorage + Supabase | 设置和知识库存本地，对话记录存云端 |

### 2.3 为什么不用后端服务器？

整个系统采用 **Serverless架构**，原因：

1. **凡科沙盒限制**：凡科网站拦截对外部域名的fetch请求，经过测试只有 Supabase REST API 和 Moonshot AI API 可以正常访问
2. **降低成本**：不需要租用云服务器，全部使用免费服务
3. **简化部署**：不需要配置Nginx、SSL、域名解析等
4. **前端直发**：客服组件直接调用AI API和数据库API，不经过中间层

### 2.4 各服务之间的数据流

```
1. 客户打开官网 → 加载 chat-widget.js
   → 自动生成访客ID（localStorage）
   → 显示AI开场欢迎语

2. 客户发送消息 → chat-widget.js
   → ① 存储到 Supabase（role='user'）
   → ② 发送到 Moonshot AI 获取回复
   → ③ 收到AI回复后存储到 Supabase（role='ai'）
   → ④ 如果是第2条用户消息 → 发送飞书通知
   → ⑤ 在聊天窗口显示AI回复

3. 管理员打开 admin.html
   → 从 Supabase 拉取所有访客对话
   → 按访客ID分组展示
   → 读取 localStorage 显示设置和知识库

4. 管理员修改设置
   → 保存到 localStorage（chat_settings）
   → 客服组件下次加载时读取同一设置
```

function getVisitorId() {
  // 1. 先从 localStorage 读取
  let id = localStorage.getItem('chat_visitor_id');
  
  // 2. 如果没有，生成新ID
  if (!id) {
    id = 'V' + Date.now().toString(36) + Math.random().toString(36).substr(2, 5);
    // 格式示例：Vlxxxxxabc123
    localStorage.setItem('chat_visitor_id', id);
  }
  
  // 3. 返回ID（同一浏览器永远不变）
  return id;
}

### 3.8 Supabase 存储逻辑

```javascript
function saveToSupabase(role, content) {
  const visitorId = getVisitorId();
  
  fetch(SUPABASE_URL + '/rest/v1/chat_records', {
    method: 'POST',
    headers: {
      'apikey': SUPABASE_KEY,
      'Authorization': 'Bearer ' + SUPABASE_KEY,
      'Content-Type': 'application/json',
      'Prefer': 'return=minimal'    // 不返回插入的数据，节省流量
    },
    body: JSON.stringify({
      visitor_id: visitorId,
      role: role,         // 'user' 或 'ai'
      content: content
    })
  });
  // 注：不等待响应，异步发送，失败不影响对话流程
}
```

### 3.9 AI对话流程（sendMessage函数）

```javascript
async function sendMessage() {
  // 1. 获取用户输入
  const text = inputEl.value.trim();
  if (!text) return;
  inputEl.value = '';
  
  // 2. 在UI显示用户消息
  addMessage('user', text);
  chatHistory.push({role:'user', content:text});
  
  // 3. 保存用户消息到Supabase
  saveToSupabase('user', text);
  
  // 4. 检查是否需要发送飞书通知
  const userMsgCount = chatHistory.filter(m => m.role === 'user').length;
  if (userMsgCount === 2) {
    sendFeishuNotification(summary);
  }
  
  // 5. 显示"正在输入..."动画
  const typingEl = addMessage('typing', '');
  sendBtn.disabled = true;
  
  // 6. 调用 Moonshot AI API
  const res = await fetch(CONFIG.apiUrl, {
    method: 'POST',
    headers: {
      'Content-Type': 'application/json',
      'Authorization': 'Bearer ' + CONFIG.apiKey
    },
    body: JSON.stringify({
      model: CONFIG.model,
      messages: [
        {role:'system', content: SYSTEM_PROMPT},  // 系统提示词
        ...chatHistory                             // 对话历史
      ],
      temperature: 0.7,    // 温度：0.7（适中创造性）
      max_tokens: 200      // 最大token数：200（约50-100个汉字）
    })
  });
  
  // 7. 解析AI回复
  const data = await res.json();
  const reply = data.choices[0].message.content;
  
  // 8. 在UI显示AI回复
  typingEl.remove();
  addMessage('ai', reply);
  chatHistory.push({role:'assistant', content:reply});
  
  // 9. 保存AI回复到Supabase
  saveToSupabase('ai', reply);
  
  // 10. 恢复输入状态
  sendBtn.disabled = false;
  inputEl.focus();
}
```

### 3.10 SYSTEM_PROMPT 设计逻辑

SYSTEM_PROMPT 是 AI 的"行为准则"，决定了AI的回复风格和内容。设计要点：

1. **身份定义**：明确告诉AI它是影月影视的客服小影
2. **业务知识**：用 `KNOWLEDGE_BASE` 变量提供公司和业务信息
3. **核心约束**：每次回复不超过50字、真人语气、诚恳热情
4. **绝对禁令**：用 `【🚫 绝对禁令】` 格式强调禁止行为
5. **回复策略**：分第1次、第2次、第3次+ 分别给出策略
6. **正确示例**：提供正确和错误的对话示例
7. **行业匹配**：提供各行业明星企业列表

## 第五部分：数据流与交互流程

### 5.1 客户首次访问流程

```
1. 客户打开官网
   ↓
2. 浏览器加载 chat-widget.js（从GitHub Pages）
   ↓
3. 组件初始化：
   a. 检查 localStorage 是否有访客ID
      - 无 → 生成新ID → 保存到 localStorage
      - 有 → 使用已有ID
   b. 创建UI（按钮+聊天窗口，窗口默认隐藏）
   ↓
4. 客户点击 💬 在线客服 按钮
   ↓
5. 聊天窗口显示，自动发送开场欢迎语（仅UI显示，不存储）
   ↓
6. 客户输入第1条消息并发送
   a. 显示在UI上
   b. 存储到 Supabase（role='user'）
   c. 调用 Moonshot AI API
   d. 收到AI回复，显示在UI上
   e. 存储AI回复到 Supabase（role='ai'）
   ↓
7. 客户发送第2条消息
   a. 同上流程（a-e）
   b. 额外：触发飞书通知（每个访客只通知一次）
   ↓
8. 客户继续对话...
   每次消息都存储到Supabase，但不再发飞书通知
```

### 5.2 管理员查看访客流程

```
1. 管理员打开 admin.html
   ↓
2. 页面加载时自动调用 loadAllData()
   a. 从 Supabase 拉取所有 chat_records（按时间倒序）
   b. 按 visitor_id 分组，构建访客列表
   c. 缓存到 cloudData 和 cloudVisitors 变量
   d. 渲染数据概览页面
   ↓
3. 管理员点击 👥 访客记录
   a. 渲染访客表格
   b. 显示访客ID、首次访问、最后活跃、消息数、最后消息
   ↓
4. 管理员点击「查看对话」
   a. 打开模态框
   b. 从 cloudVisitors[vid] 获取该访客所有消息
   c. 按时间顺序渲染对话气泡
```

### 5.3 通知开关的工作流程

```
1. 管理员打开 admin.html → 系统设置
   ↓
2. renderSettings() 被调用
   a. 从 localStorage 读取 chat_settings
   b. 设置飞书开关的 checked 状态
   ↓
3. 管理员切换开关
   a. onchange 触发 saveNotifySettings()
   b. 将新状态保存到 localStorage.chat_settings.notifyFeishu
   c. 显示 "通知设置已保存" Toast
   ↓
4. 客户端（凡科网站）加载 chat-widget.js
   a. sendFeishuNotification() 被调用时
   b. 从 localStorage 读取 chat_settings.notifyFeishu
   c. 如果为 false → 直接返回，不发送通知
   d. 如果为 true（或未设置，默认true）→ 发送通知
   
⚠️ 注意：开关状态通过 localStorage 共享
   后台和客服组件需要在同一浏览器中才能同步
   跨浏览器需要手动保持设置一致
```

## 第八部分：AI回复策略与Prompt设计

### 8.1 当前策略（v4.2）

**规则1：前两次回复绝对不能提微信**
- 第1次回复：只回答客户问题 + 展示公司实力
- 第2次回复：继续解答 + 可以问客户需求
- 绝对禁止在第1、2句出现"微信""电话""加好友""联系方式""导演微信"等

**规则2：第三次回复以后，才能提一次微信引导**
- 如果前两次都没提微信，第3次及以后可以说一次引导话术
- 如果已经引导过微信，绝对不能再提第二次

**规则3：一次对话只能引导一次加微信**

**规则4：展示公司实力**
- 服务过500多家上市公司
- 合作客户包括：华为、腾讯、阿里巴巴、比亚迪、美的、格力、顺丰、万科、海底捞、小米

**规则5：不提具体价格**
- 客户问价格时，回答"都是定制的，看具体需求"

**规则6：每次回复不超过50个字**

### 8.2 微信询问话术
```
对了，聊了这么多还没请教您怎么称呼？方便留个微信或手机号吗？
我把您的需求跟导演说一下，让导演直接加您沟通~
```

### 8.3 Prompt设计要点

一个好的 SYSTEM_PROMPT 应该包含：

1. **身份定义**：你是谁，代表哪家公司
2. **业务知识**：公司产品、服务、优势、价格策略
3. **约束条件**：字数限制、语气要求、禁止事项
4. **回复策略**：不同轮次的回复策略
5. **示例对话**：正确和错误的示例
6. **变量替换**：如 `{wxNumber}` 等可配置项
